# Smart Travel  慧行

#### 介绍
开发本作品旨在解决视障人士出行、环境感知、实时求助等核心痛点，弥补现有导盲类产品功能单一、服务体系不完善的不足。市面同类竞品主要为纯导航类（百度地图盲人版、腾讯地图无障碍模式）、单一感知类（普通智能盲杖、智能识别眼镜）、简易求助类（公益帮扶小程序），本系统对标此类产品进行全方位功能升级与体系搭建，打造更贴合视障人士全场景需求的智能导盲服务平台。

#### 软件架构
软件架构说明
1 系统整体架构
本系统采用“终端层-通信层-服务层-数据层”的四层分层架构，实现盲人端、志愿者端与服务端的一体化协同运行，层级清晰、职责明确，有效降低模块耦合度，提升可扩展性与可维护性，如图2.1所示。该架构高内聚、低耦合，各层可独立迭代优化，便于后续功能拓展与性能调优，同时通过多协议通信与分层解耦，保障系统在高并发场景下的稳定性与可靠性，为视障人士提供持续、安全的服务支撑。

#### 功能模块1划分
系统按盲人端、志愿者端、服务端三大主体划分功能模块，各主体模块并非独立运行，而是通过标准化接口实现高效、精准的数据交互，形成协同联动的完整体系。盲人端聚焦盲人用户的实际使用需求，核心围绕求助、感知、安全等核心场景设计功能模块，具体包括：
1.	求助交互：语音通话、视频通话、求助发起和取消、求助状态展示；
2.	环境感知：传感器数据采集、障碍物检测、地形分析、文本识别、语音反馈；
3.	安全监测：跌倒检测、GPS定位、警报推送、紧急联系人管理；
4.	社区交互：语音发布内容、内容语音播报、互动交流；
5.	设备管理：设备连接、续航监测、参数调试、异常提醒；
6.	个人中心：个人信息维护。
志愿者端主要服务于志愿者开展帮扶工作，围绕接单、响应、成长等场景设计，与盲人端形成精准联动，具体模块包括：
1.	求助响应模块：语音或者视频通话、实时交互；
2.	社区模块：内容发布、内容查看、互动交流；
3.	成长中心模块：服务数据统计、经验值累计、勋章解锁、积分获取；
4.	培训中心模块：课程学习、在线考核、学习记录；
5.	服务记录模块：帮助统计、历史记录、详情查看；
6.	积分商城模块：积分展示、商品兑换、兑换记录；
7.	个人中心模块：个人信息、设置、消息通知。
服务端作为整个系统的核心支撑，承担着数据处理、调度管理、安全保障等关键职责，为盲人端与志愿者端提供全方位支持，具体功能模块如下：
1.	求助调度模块：求助信息转发、互斥接收机制实现、接单状态同步；
2.	社区管理模块：内容审核、发布或者查看接口、互动数据处理；
3.	成长体系模块：经验值、积分和勋章计算，服务数据统计等；
4.	设备管理模块：终端设备状态监控、数据同步、异常预警；
5.	缓存与存储模块：热点数据缓存、关系型数据存储、文件存储。

其中，盲人端遵循“语音优先的非视觉交互（VUI）”原则，无复杂图形界面，所有操作通过语音指令、物理按键实现，反馈形式为语音播报和触觉震动，界面简洁，无视觉信息冗余；志愿者端遵循简洁高效、视觉清晰原则，接单大厅突出求助核心信息，操作按钮大且醒目，数据统计可视化展示，支持多形式内容交互；所有界面均符合无障碍设计规范，支持字体放大、高对比度、语音读屏，操作流程简化，避免多步骤复杂操作。
